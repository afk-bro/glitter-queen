# UI Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Improve contrast/readability, CTA button hierarchy, hero visual punch, and product card hover interaction.

**Architecture:** CSS token additions feed into CVA variant changes and component updates. Tasks are ordered so each depends only on what came before: tokens first, then components that consume them, then call sites. `QuickViewButton` is extracted as a client component to keep `ProductCard` server-rendered.

**Tech Stack:** Next.js 16 App Router, Tailwind v4 (CSS-first, no config file), shadcn/ui CVA buttons, Vitest + Testing Library

In `app/globals.css`, inside `:root { }`, add after `--primary-soft`:

```css
  --primary-vivid:       #6d28d9;
  --glow-cta:           color-mix(in srgb, var(--primary-vivid) 35%, transparent);
  --glow-cta-strong:    color-mix(in srgb, var(--primary-vivid) 50%, transparent);
  --glow-cta-soft:      color-mix(in srgb, var(--primary-vivid) 25%, transparent);
```

- [ ] **Step 2: Add dark-mode override**

Inside `.dark { }`, add after `--primary-soft`:


This makes `bg-primary-vivid`, `text-primary-vivid`, `ring-primary-vivid` etc. available as Tailwind utilities.

## Task 7: Create QuickViewButton client component

**Files:**
- Create: `components/shop/QuickViewButton.tsx`
- Create: `__tests__/components/QuickViewButton.test.tsx`

- [ ] **Step 1: Write failing tests**

Create `__tests__/components/QuickViewButton.test.tsx`:

```tsx
import { render, screen, fireEvent } from '@testing-library/react'
import { describe, it, expect, vi, beforeEach } from 'vitest'

const mockToast = vi.fn()
vi.mock('sonner', () => ({ toast: mockToast }))

import { QuickViewButton } from '@/components/shop/QuickViewButton'

describe('QuickViewButton', () => {
  beforeEach(() => {
    mockToast.mockClear()
  })

  it('renders a button with text "Quick view"', () => {
    render(<QuickViewButton />)
    expect(screen.getByRole('button', { name: 'Quick view' })).toBeInTheDocument()
  })

  it('calls toast with correct message on click', () => {
    render(<QuickViewButton />)
    fireEvent.click(screen.getByRole('button', { name: 'Quick view' }))
    expect(mockToast).toHaveBeenCalledWith('Quick view coming soon')
  })

  it('prevents default on click to avoid navigating the parent link', () => {
    render(<QuickViewButton />)
    const btn = screen.getByRole('button', { name: 'Quick view' })
    const event = new MouseEvent('click', { bubbles: true, cancelable: true })
    btn.dispatchEvent(event)
    // preventDefault called by the handler stops navigation
    expect(event.defaultPrevented).toBe(true)
  })
})
```

- [ ] **Step 2: Run to verify they fail**

```bash
npx vitest run __tests__/components/QuickViewButton.test.tsx
```

Expected: all 3 fail — module not found.

- [ ] **Step 3: Create the component**

Create `components/shop/QuickViewButton.tsx`:

```tsx
'use client'

import { toast } from 'sonner'

export function QuickViewButton() {
  return (
    <button
      type="button"
      onClick={(e) => {
        e.preventDefault()
        toast('Quick view coming soon')
      }}
      className="pointer-events-auto rounded-full bg-white/90 text-foreground text-xs font-body px-4 py-1.5 hover:bg-white transition-colors duration-300"
      aria-label="Quick view"
    >
      Quick view
    </button>
  )
}
```

`e.preventDefault()` stops the parent `<Link>` from navigating when the button is clicked.
